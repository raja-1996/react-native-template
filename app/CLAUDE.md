# app
Expo (React Native) frontend — project root containing config files and the src/ source tree.

- `app.json` — Expo app configuration: scheme, plugins (expo-router, expo-secure-store, expo-image-picker, expo-image), typedRoutes experiment, bundle identifiers for iOS/Android
  - gotcha: `newArchEnabled: true` — React Native new architecture is ON; some third-party libs may be incompatible

- `package.json` — dependency manifest; main entry is `expo-router/entry`
  - deps: expo ~55, react-native 0.79.2, expo-router ~4.0, @tanstack/react-query ^5, zustand ^5, @supabase/supabase-js ^2, axios ^1.7
  - test deps: jest-expo, @testing-library/react-native ^13
  - gotcha: test runner is `jest` with `jest-expo` preset

- `tsconfig.json` — TypeScript config extending `expo/tsconfig.base`; path alias `@/*` maps to `./src/*`
  - gotcha: `@/` alias resolves to `src/` — use for all non-relative imports within src/

- `jest.config.js` — Jest config using `jest-expo` preset
  - gotcha: `transformIgnorePatterns` explicitly includes `zustand` and all expo packages — required for ESM modules; do NOT simplify this pattern

- `expo-env.d.ts` — Expo environment type declarations (generated by Expo, do not edit)

- `src/` — all application source code (screens, components, stores, hooks, services, lib, constants, tests)
